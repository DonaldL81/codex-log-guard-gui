# codex写盘异常检测

一个 Windows 小工具，用来检查和缓解 Codex 本地日志异常写盘问题。

它可以查看 `logs_2.sqlite` 写盘状态、安装日志拦截保护、清理日志文件、实时监测写盘强度，并保存监测记录。

## 问题背景

部分 Codex 用户遇到过本地日志持续写入问题，主要涉及：

```text
C:\Users\<用户名>\.codex\logs_2.sqlite
```

常见表现：

- Codex 空闲或运行任务时仍有持续磁盘写入。
- 电脑卡顿、会话切换变慢、响应延迟。
- 主数据库文件增长不明显，但 SQLite WAL 文件可能持续刷写。
- 日志主要是底层运行和调试记录，不是用户对话正文。

这类持续写入可能消耗 SSD 写入寿命。具体写入速度和累计写入量会因 Codex 版本、系统环境和使用方式不同而变化，但如果怀疑 `logs_2.sqlite` 异常增长，就值得检查和临时止血。

本工具是临时缓解和观测工具，不是 Codex 官方补丁，也不修复 Codex 内部日志逻辑。

## 下载后怎么用

双击启动：

```text
codex写盘异常检测.vbs
```

如果已经打开一个窗口，再次双击会在原窗口提示是否关闭并重新打开。

推荐流程：

1. 先完全退出 Codex。
2. 打开本工具。
3. 查看“Codex 日志写盘文件”的大小和日志行数。
4. 如日志文件很大，点击 `清理文件`。
5. 如显示 `拦截器：未安装`，点击红色 `拦截器未安装` 安装保护。
6. 重新打开 Codex。
7. 点击 `开始监测`。
8. 在 Codex 中运行一个实际任务，观察写盘评估和监测明细。
9. 监测结束后点击 `停止监测`。

## 界面怎么看

### 写盘评估

顶部是最重要的结论区。

| 状态 | 含义 |
|---|---|
| 未检测 | 还没有开始监测 |
| 观察中 | 样本不足，暂时不下结论 |
| 正常 | 近一段时间写盘较低 |
| 少量偏高 | 有一定写盘，但未达到异常阈值 |
| 异常偏高 | 写盘超过异常阈值，且没有被拦截保护覆盖 |
| 少量偏高已拦截 | 发现少量日志写入尝试，已被拦截 |
| 异常偏高已拦截 | 发现较多日志写入尝试，已被拦截 |

颜色含义：

- 红色：需要关注，例如 `异常偏高`。
- 橙色：轻微偏高或观察中。
- 绿色：当前正常。
- 蓝色：异常写入已被拦截。

### 防护状态

这里显示拦截保护和计数设置。

| 显示 | 说明 |
|---|---|
| 拦截器：未安装 | 当前没有日志拦截保护 |
| 拦截器：拦截保护中 | 当前已安装日志拦截保护 |
| 历史累计拦截 | 历史累计拦截次数 |
| 上轮新增拦截 | 上一轮计数周期新增次数 |
| 监测时计数 | 只有开始监测时才会临时统计拦截次数 |
| 监测时不计数 | 开始监测时只看写盘，不统计拦截次数 |

注意：

- 安装拦截器后，平时是低写入的纯拦截状态。
- `监测时计数` 只是偏好设置，不代表平时一直计数。
- 关闭 `监测时计数` 会清空历史累计拦截次数。

### 文件管理

这里用于查看和清理 Codex 日志文件。

| 操作 | 说明 |
|---|---|
| 打开目录 | 打开 `.codex` 日志目录 |
| 清理文件 | 清空上一次备份，并把当前 `logs_2.sqlite*` 移动到 `logs_backup` |
| 清空历史 | 只清空 `logs_backup`，不处理当前日志文件 |

清理当前日志文件前请先完全退出 Codex。工具不会强制结束 Codex 进程。

### 实时监测和监测明细

开始监测后，会显示：

- 总写入 MB/s
- 最高写入进程
- 当前会话
- 本次采样拦截
- 本轮累计拦截
- 本次监测日志路径

下方表格按时间倒序显示采样记录，最新记录在最上方。

## 重要说明

- 本工具仅适用于 Windows。
- 需要系统中可用的 `python` 或 `py -3`，用于读取和修改 SQLite。
- 监测记录保存在工具目录下的 `monitor-logs`。
- 监测记录可能包含会话标题、会话 ID、工作目录和进程信息。
- 工具不读取用户消息正文、Codex 回复正文、API Key 或 Token。
- 写盘速率来自 Windows 性能计数器；如果系统性能计数器异常，监测界面可能短时间无响应，重启系统通常可以恢复。

## 文件位置

| 内容 | 默认位置 |
|---|---|
| Codex 日志写盘文件 | `C:\Users\<用户名>\.codex\logs_2.sqlite` |
| 日志备份暂存目录 | `C:\Users\<用户名>\.codex\logs_backup` |
| 本工具监测记录 | `monitor-logs` |
| 启动入口 | `codex写盘异常检测.vbs` |
| 主程序 | `CodexLogGuard.ps1` |

## 自检

如果工具无法正常启动，可以在 PowerShell 中运行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\CodexLogGuard.ps1 -SelfTest
```

看到下面输出表示基础链路正常：

```text
SelfTest OK
```
